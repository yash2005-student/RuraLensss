# GNN Infrastructure Impact Prediction Model

Version: 1.0
Date: April 17, 2026
Framework: PyTorch + PyTorch Geometric

## 1. Overview

This model predicts cascading infrastructure impact on graphs that represent village digital twin assets.

- Input per node: 24 features
- Output per node: 12 impact logits (converted to probabilities with sigmoid)
- Graph type: directed or mixed directed/undirected
- Edge strength: supported through edge weights

The implementation is Python-only and can run directly in Gradio without frontend or JavaScript integration.

### 1.1 Model Currently Used (Code-Accurate)

Current runtime and training setup from model.py and train.py:

- Core model class: InfrastructureGNN
- Predictor wrapper: ImpactPredictor
- Inference API loader: api_server.py loads models/gnn_model.pt when present
- Default training objective in use: weighted BCEWithLogitsLoss (FocalLoss is optional and OFF by default)
- Default training procedure in use: 5-fold cross validation on synthetic graph dataset

## 2. Architecture

Implemented in model.py as InfrastructureGNN:

- Layer 1: GCNConv(24 -> 128), ReLU, Dropout(0.3)
- Layer 2: GATConv(128 -> 128 via 4 heads x 32, edge_dim=1), residual from Layer 1, ReLU, Dropout(0.3)
- Layer 3: GCNConv(128 -> 128), residual from Layer 1, ReLU, Dropout(0.3)
- Layer 4: Linear(128 -> 12), no output activation

Inference probabilities are computed as:

- probabilities = sigmoid(logits / temperature)

## 3. Node Features (24)

Feature layout:

- 0-11: one-hot infrastructure type
- 12-23: operational features

Operational ordering in this code:

- 12 capacity
- 13 current_level
- 14 flow_rate
- 15 status
- 16 criticality
- 17 population_served
- 18 economic_value
- 19 connectivity
- 20 maintenance_score
- 21 weather_risk
- 22 failure_history
- 23 reserved

## 4. Synthetic Dataset Generation

Implemented in train.py:

- Total default samples: 1000
- Node count per graph: uniform random in [10, 30]
- Sparse topology with average 3-5 neighbors per node
- Edge weights sampled in [0.5, 1.0]

Ground-truth impact labels are generated by BFS-style cascade:

1. Pick a random failure node.
2. Assign high initial impact on that node.
3. Propagate to neighbors with:
   - depth decay
   - edge-quality modulation
   - random attenuation
4. Add Gaussian noise N(0, 0.05).
5. Clip labels to [0, 1].

Node features are standardized per graph with z-score normalization and clipped to [-3, 3].

## 5. Training Configuration

Implemented in model.py and train.py:

- Epochs: 50 (default)
- Batch size: 32 (default)
- Optimizer: Adam(lr=0.001, betas=(0.9, 0.999), eps=1e-8)
- Weight decay: 0.0
- Scheduler: ReduceLROnPlateau(mode=min, factor=0.5, patience=5, threshold=1e-4, min_lr=1e-6)
- Gradient clipping: max_norm=1.0

Loss:

- BCEWithLogitsLoss(reduction=none)
- Critical-node weighting: 3x where target mean impact > 0.5

### 5.1 Exact Loss Used By Default

The default predictor initialization is:

- use_focal_loss=False

So the active objective is weighted BCEWithLogitsLoss:

1. Base per-node, per-output loss is computed with BCEWithLogitsLoss(reduction=none).
2. A weight mask is applied:
   - weight = 3.0 for nodes where mean(targets over 12 outputs) > 0.5
   - weight = 1.0 otherwise
3. Final scalar loss is mean(loss_per_node * weights).

In formula form:

- L = mean(BCEWithLogits(logits, targets) * w)
- w_i = 3.0 if mean(target_i) > 0.5, else 1.0

## 6. Five-Fold Cross Validation

Implemented in train.py as train_model_with_five_fold_cv:

- 5 folds with shuffled KFold split
- Per fold:
  - Train for configured epochs
  - Track best validation loss
  - Save fold checkpoint to models/cv/gnn_model_fold{n}.pt
- Global best fold checkpoint is copied to:
  - models/gnn_model.pt

CV summary is saved to:

- models/cv/cv_results.json

### 6.1 Training Data vs Validation Data (Current Defaults)

Data source and split strategy currently used by train.py:

- Dataset type: synthetic graphs generated by generate_training_data
- Default total graphs: 1000
- Split method: KFold(n_splits=5, shuffle=True, random_state=42)
- Per fold (with default 1000 samples):
   - Training graphs: 800
   - Validation graphs: 200
- Every graph contains node-level 24D features and 12D labels.

This means each fold trains on 80% of generated data and validates on 20%, then selects the best checkpoint by minimum validation loss.

## 7. Cumulative Multi-Failure Simulation

Implemented in simulation_engine.py:

- run_simulation: one-shot failure set
- run_cumulative_simulation: stepwise stacked failures

Example sequence [0, 2, 1]:

- Step 1: fail node 0
- Step 2: fail nodes 0 + 2
- Step 3: fail nodes 0 + 2 + 1

Per step, the engine reports:

- cumulative delta versus healthy baseline
- incremental delta versus previous step

This enables node-after-node effect analysis exactly for operational what-if workflows.

## 8. Gradio Usage (Python-Only)

Interface file: gradio_app.py

Features:

- Prediction tab for custom graph input
- What-If tab for delta-inference simulation
- Cumulative sequence section for stacked failures

Run commands (from backend/python-gnn):

1. Train model with five-fold CV:

   python train.py --epochs 50 --batch-size 32 --samples 1000

2. Launch Gradio:

   python gradio_app.py

Default Gradio URL:

- http://localhost:7875
